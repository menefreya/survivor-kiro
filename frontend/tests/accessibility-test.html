Purpose: This page helps you manually test the accessibility features implemented in the Survivor Fantasy League application.

1. Keyboard Navigation Test

Test keyboard navigation throughout the application:

  • Press Tab on page load - Skip link should appear
  • Press Tab repeatedly - Focus should move through all interactive elements
  • Press Shift + Tab - Focus should move backwards
  • Press Enter on buttons - Should activate the button
  • Press Space on buttons - Should activate the button
  • Navigate forms using Tab - Should move between fields
  • Use Arrow keys in dropdowns - Should change selection
Test Element: Click here and press Tab to test focus indicator

2. Focus Indicator Test

Verify all interactive elements have visible focus states:

  • Buttons show 3px orange outline on focus
  • Links show 3px orange outline on focus
  • Form inputs show 3px orange outline on focus
  • Navigation links show white outline on focus
  • Draggable items show outline on focus
  • Cards show outline when focused within

4. Screen Reader Test

Test with a screen reader (VoiceOver on Mac, NVDA on Windows):

VoiceOver (Mac): Press Cmd + F5 to enable
NVDA (Windows): Download from nvaccess.org
  • All images are announced with descriptive alt text
  • Form labels are read when focusing inputs
  • Error messages are announced immediately
  • Success messages are announced politely
  • Loading states are announced
  • Button purposes are clear
  • Navigation structure is logical
  • Headings create proper hierarchy

5. Reduced Motion Test

Enable reduced motion in your operating system:

macOS: System Preferences ‚Üí Accessibility ‚Üí Display ‚Üí Reduce motion
Windows: Settings ‚Üí Ease of Access ‚Üí Display ‚Üí Show animations (turn off)
  • Animations are disabled or minimal
  • Transitions are instant or very short
  • Hover effects don't use transforms
  • Loading spinners are static or minimal
  • Functionality still works correctly

6. High Contrast Test

Enable high contrast mode in your operating system:

macOS: System Preferences ‚Üí Accessibility ‚Üí Display ‚Üí Increase contrast
Windows: Settings ‚Üí Ease of Access ‚Üí High contrast
  • All text remains readable
  • Borders are visible on buttons and cards
  • Focus indicators are clearly visible
  • Form inputs have visible borders
  • Interactive elements are distinguishable

7. Mobile Touch Target Test

Test on mobile device or use browser DevTools mobile emulation:

  • All buttons are at least 44x44px
  • Links are easy to tap
  • Form inputs are large enough
  • No accidental taps on nearby elements
  • Spacing between interactive elements is adequate

8. Automated Testing

Run automated accessibility tests:

Lighthouse Audit

  1. Open Chrome DevTools (F12)
  2. Go to "Lighthouse" tab
  3. Select "Accessibility" category
  4. Click "Generate report"
  5. Target Score: 95+ (100 is ideal)

axe DevTools

  1. Install axe DevTools extension
  2. Open DevTools
  3. Go to "axe DevTools" tab
  4. Click "Scan ALL of my page"
  5. Fix any critical or serious issues

9. ARIA Attributes Test

Verify ARIA attributes are correctly implemented:

  • Required fields have aria-required="true"
  • Invalid fields have aria-invalid="true"
  • Error messages linked with aria-describedby
  • Loading buttons have aria-busy="true"
  • Current page has aria-current="page"
  • Regions have aria-label or aria-labelledby
  • Live regions use aria-live="polite" or role="alert"

10. Semantic HTML Test

Verify proper HTML structure:

  • Page has single <h1> element
  • Headings follow logical hierarchy (h1 ‚Üí h2 ‚Üí h3)
  • Navigation uses <nav> element
  • Main content uses <main> element
  • Sections use <section> or <article>
  • Lists use <ul>, <ol>, or <dl>
  • Tables use <th> with scope attribute
  • Forms use <label> elements
